# SwarmConductor

> **SwarmConductor** — это архитектурный фреймворк промптов для управления ИИ-агентами-разработчиками. 
> Он физически запрещает базовой LLM писать код напрямую, превращая её из хаотичного "кодера-одиночки" в строгого Техлида (Principal Engineer), управляющего пулом из 7 субагентов.

---

## Проблема и Решение

**Уязвимость базовых LLM:** При запросе на исправление бага языковые модели склонны "срезать углы". Они пишут весь код в одном контексте, игнорируют поиск `best-practices`, выдумывают несуществующие инструменты и пропускают этап код-ревью.

**Решение: Gatekeeper Protocol**
Фреймворк внедряет жесткий 4-ступенчатый пайплайн, блокирующий завершение задачи до прохождения аудита:

`Разведка` ➔ `Планирование` ➔ `Исполнение` ➔ `Критический барьер (QA)`

## Структура Репозитория

В зависимости от физического "движка" вашей среды разработки, используйте соответствующую директорию:

| Среда | Директория | Особенности архитектуры |
| :--- | :--- | :--- |
| **Antigravity IDE** | `/antigravity-ide` | Нативная поддержка параллельных агентов, использование флагов `enable_subagent_tools: true`, запись логов в системную директорию `brain/`. |
| **Claude Code, Cursor** | `/universal-agents` | Адаптация под линейный CLI-интерфейс. Эмуляция роя через явное объявление ролей в консоли. Хранение артефактов в локальной папке `.swarm-artifacts/`. |

## Пул Субагентов (Staff Pool)

Главному агенту разрешено делегировать задачи *исключительно* следующим профилям (любые галлюцинации и выдумывание новых ролей заблокированы):

1. **`asw-explorer`** : Разведка, Web-поиск и аудит текущего дизайн-кода.
2. **`asw-planner`** : Декомпозиция, генерация `task.md` и оценка рисков.
3. **`asw-architect`** : Написание бизнес-логики, TypeScript, роутинга.
4. **`asw-designer`** : Верстка UI с жестким сохранением текущего стиля проекта.
5. **`asw-critic`** : *"Адвокат Дьявола"*. Код-ревью, поиск утечек памяти и ререндеров.
6. **`asw-security-expert`** : Аудит уязвимостей, SQLi, политик базы данных.
7. **`asw-tester`** : Написание и прогон E2E / Unit тестов.

### Для Antigravity IDE
```text
1. Скопируйте содержимое master-prompt.md в глобальные настройки (System Persona) или в файл .antigravityrules в корне проекта.
