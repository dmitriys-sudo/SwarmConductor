# SwarmConductor

> **SwarmConductor** — это архитектурный фреймворк промптов для управления ИИ-агентами-разработчиками. 
> Он физически запрещает базовой LLM писать код напрямую, превращая её из хаотичного "кодера-одиночки" в строгого Техлида (Principal Engineer), управляющего пулом из 7 субагентов.

---

## Проблема и Решение

**Уязвимость базовых LLM:** При запросе на исправление бага языковые модели склонны "срезать углы". Они пытаются написать весь код в одном контексте, игнорируют поиск `best-practices`, выдумывают несуществующие инструменты и пропускают этап код-ревью.

**Решение: Gatekeeper Protocol**
Фреймворк внедряет жесткий 4-ступенчатый пайплайн, блокирующий завершение задачи до прохождения аудита:

`Разведка` ➔ `Планирование` ➔ `Исполнение` ➔ `Критический барьер (QA)`

---

## Поддерживаемые Среды

Структура репозитория разделена на две версии под разные архитектуры:

* `/antigravity-ide` — нативная версия для среды Antigravity с поддержкой фоновых процессов и системных триггеров.
* `/universal-agents` — универсальная версия для CLI-инструментов (Claude Code, Cursor, Aider). Эмулирует рой в линейном чате и автоматически управляет локальным окружением.

---

## Пул Субагентов (Staff Pool)

Главному агенту разрешено делегировать задачи *исключительно* следующим профилям (любые галлюцинации и выдумывание новых ролей заблокированы):

| Агент | Зона ответственности | Роль в пайплайне |
| :--- | :--- | :--- |
| **`asw-explorer`** | Разведка, Web-поиск и аудит текущего дизайн-кода. | Ступень 1 |
| **`asw-planner`** | Декомпозиция, генерация `task.md` и оценка рисков. | Ступень 2 |
| **`asw-architect`** | Написание бизнес-логики, TypeScript, роутинга. | Ступень 3 |
| **`asw-designer`** | Верстка UI с жестким сохранением текущего стиля проекта. | Ступень 3 |
| **`asw-critic`** | Код-ревью, поиск утечек памяти и ререндеров. | Ступень 4 |
| **`asw-security-expert`** | Аудит уязвимостей, SQLi, политик базы данных. | Ступень 4 |
| **`asw-tester`** | Написание и прогон E2E / Unit тестов. | Ступень 4 |

## Установка (Installation)

Архитектура состоит из двух ключевых компонентов: **Мастер-промпта** (личность и ограничения) и **Скилла** (алгоритм работы роя). Выберите нужную версию (`antigravity-ide` или `universal-agents`) и выполните следующие шаги:

### 1. Установка Мастер-промпта
Файл: `master-prompt.md`

Этот файл задает базовые ограничения для среды и превращает ИИ в координатора. 
* **Инструкция:** Установите содержимое этого файла нативно в вашей рабочей среде как главный системный промпт (например, в настройки *System Persona*, *AI Rules* или конфигурационный файл вроде `.antigravityrules` / `.cursorrules`). Точный метод внедрения зависит от вашей среды.

### 2. Установка Скилла
Файл: `swarm-conductor.md`

Этот файл содержит логику ступенчатого пайплайна и список доступных субагентов.
* **Инструкция:** Скопируйте абсолютный путь к файлу `swarm-conductor.md` на вашем компьютере. Перейдите в чат или терминал вашей среды разработки и отправьте агенту команду:

```text
Установи этот файл как новый скилл. Путь: [ВСТАВЬТЕ_ВАШ_ПУТЬ_СЮДА]
